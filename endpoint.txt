# Подробная документация REST API эндпоинтов (Fuel Flow Control)

Эта документация описывает все API-ручки фронтенда, обеспечивающие полную интерактивность приложения. Backend на Python (FastAPI/Flask) должен реализовывать аналогичный набор эндпоинтов для обеспечения бесшовного сопряжения с клиентом.

---

## 1. АЗС (Gas Stations)

### GET /api/stations
* **Описание**: Получение списка всех зафиксированных автозаправочных станций (АЗС) с текущими статусами наличия топлива, очередями, координатами и метаданными.
* **Ответ**: `Array<GasStation>`
  ```json
  [
    {
      "id": "station-real-1",
      "brand": "Роснефть",
      "address": "г. Анапа, Анапское шоссе, 1",
      "coords": { "lat": 44.8952, "lng": 37.3165 },
      "status": {
        "AI92": "available",
        "AI95": "available",
        "DT": "empty"
      },
      "queueLength": "medium",
      "waitingTime": 20,
      "confidenceScore": 0.85,
      "deficitForecast": "medium",
      "nextDelivery": "нет данных",
      "updatedAt": "2026-07-17T17:42:00Z",
      "apiUrl": "https://api.example.com/sync",
      "apiKey": "secret_key"
    }
  ]
  ```

### POST /api/stations
* **Описание**: Добавление новой АЗС через административную панель.
* **Тело запроса**:
  ```json
  {
    "brand": "Газпромнефть",
    "address": "г. Анапа, Супсехское шоссе, 4",
    "coords": { "lat": 44.8725, "lng": 37.3412 },
    "status": {
      "AI92": "available",
      "AI95": "available",
      "DT": "available"
    },
    "queueLength": "none",
    "waitingTime": 0,
    "confidenceScore": 1.0,
    "deficitForecast": "low",
    "nextDelivery": "нет данных",
    "apiUrl": "",
    "apiKey": ""
  }
  ```
* **Ответ**: `{ "status": "ok", "station": GasStation }`

### POST /api/stations/edit
* **Описание**: Полное редактирование метаданных и состояния АЗС.
* **Тело запроса**:
  ```json
  {
    "id": "station-real-1",
    "brand": "Роснефть-Изменение",
    "address": "г. Анапа, Анапское шоссе, 1а",
    "coords": { "lat": 44.8952, "lng": 37.3165 },
    "status": { "AI92": "available", "AI95": "empty", "DT": "unloading" },
    "queueLength": "short",
    "waitingTime": 10,
    "confidenceScore": 0.95,
    "deficitForecast": "medium",
    "nextDelivery": "через 2 часа",
    "apiUrl": "https://new-api.com",
    "apiKey": "new_key"
  }
  ```
* **Ответ**: `{ "status": "ok", "station": GasStation }`

### POST /api/stations/delete
* **Описание**: Удаление АЗС по ID.
* **Тело запроса**: `{ "id": "station-real-1" }`
* **Ответ**: `{ "status": "ok" }`

### POST /api/stations/override
* **Описание**: Экспресс-корректировка статусов топлива и параметров очереди (в обход ИИ-анализа со 100% уровнем доверия).
* **Тело запроса**:
  ```json
  {
    "id": "station-real-1",
    "status": {
      "AI95": "available"
    },
    "queueLength": "long",
    "waitingTime": 50,
    "deficitForecast": "high",
    "nextDelivery": "завтра утром"
  }
  ```
* **Ответ**: `{ "status": "ok", "station": GasStation }`

### POST /api/stations/sync-api
* **Описание**: Принудительный опрос внешнего интеграционного API, прописанного у АЗС, для автоматического импорта данных.
* **Тело запроса**: `{ "id": "station-real-1" }`
* **Ответ**: `{ "status": "ok", "station": GasStation }`

### POST /api/stations/refresh-analysis
* **Описание**: Запуск принудительного повторного анализа логов сообщений чатов алгоритмами ИИ для корректировки индекса доверия и статусов конкретной АЗС.
* **Тело запроса**: `{ "id": "station-real-1" }`
* **Ответ**: `{ "status": "ok", "station": GasStation }`

## 2. Интеграция и парсинг чатов (Telegram / Max Integration)

### GET /api/chats
* **Описание**: Получение списка последних 20 зафиксированных сообщений.
* **Ответ**: `Array<ChatMessage>`
  ```json
  [
    {
      "id": "msg-12345",
      "sender": "Александр К.",
      "text": "На Лукойле на Анапском шоссе завезли 95 бензин! Очереди почти нет.",
      "source": "telegram",
      "isFake": false,
      "parsed": {
        "isFuelRelated": true,
        "brand": "Лукойл",
        "address": "Анапское шоссе",
        "fuelType": "AI95",
        "status": "available",
        "queue": "short",
        "isFake": false,
        "explanation": "Разобрано NLP: подтверждено наличие АИ-95"
      },
      "createdAt": "2026-07-17T17:40:00Z"
    }
  ]
  ```

### POST /api/chats/delete
* **Описание**: Удаление отдельного сообщения по ID из истории чата.
* **Тело запроса**: `{ "id": "msg-12345" }`
* **Ответ**: `{ "status": "ok", "chats": Array<ChatMessage> }`

### POST /api/chats/clear
* **Описание**: Полное удаление всех сообщений чата (очистка лога).
* **Тело запроса**: пустой запрос `{}`
* **Ответ**: `{ "status": "ok", "chats": [] }`

### GET /api/chats/connected
* **Описание**: Проверка подключенных внешних источников (мостов). Возвращает текущий активный и все когда-либо подключенные чаты.
* **Ответ**:
  ```json
  {
    "connectedChat": {
      "id": "chat-max",
      "url": "https://max.ru/anapa-fuel",
      "title": "ТОПЛИВО СЕЙЧАС АНАПА (MAX)",
      "status": "connected",
      "connectedAt": "2026-07-17T17:30:00Z"
    },
    "connectedChats": [...]
  }
  ```

### POST /api/chats/connect-link
* **Описание**: Подключение веб-ссылки внешнего чата (t.me/... или max.ru/...). Сервер осуществляет парсинг (Scraping) последних записей, извлекает топливные сообщения и обновляет связанные станции Анапы.
* **Тело запроса**: `{ "url": "https://t.me/anapa_fuel_now" }`
* **Ответ**:
  ```json
  {
    "status": "ok",
    "connectedChat": { ... },
    "connectedChats": [ ... ],
    "channels": [ ... ],
    "stations": [ ... ]
  }
  ```

### POST /api/chats/disconnect
* **Описание**: Отключение внешнего чата по URL. Если URL не указан, отключаются все чаты.
* **Тело запроса**: `{ "url": "https://t.me/anapa_fuel_now" }`
* **Ответ**: `{ "status": "ok", "channels": [...], "connectedChat": null, "connectedChats": [] }`

### POST /api/chat/parse
* **Описание**: Основной NLP ИИ-эндпоинт. Получает неструктурированное текстовое сообщение, отправляет в Google Gemini API (с использованием настроенных System Prompt и RAG-контекста), извлекает сущности, определяет, является ли сообщение фейком, создаёт запись в логе и автоматически корректирует статус соответствующей АЗС в базе.
* **Тело запроса**:
  ```json
  {
    "text": "На Татнефти по ул. Ленина пропал дизель, сказали бензовоз только завтра будет",
    "sender": "Тимофей П."
  }
  ```
* **Ответ**:
  ```json
  {
    "status": "ok",
    "parsed": {
      "isFuelRelated": true,
      "brand": "Татнефть",
      "address": "ул. Ленина",
      "fuelType": "DT",
      "status": "empty",
      "queue": null,
      "isFake": false,
      "explanation": "Сообщение о дефиците ДТ"
    },
    "message": { "id": "msg-999", ... },
    "stations": [ ... ]
  }
  ```

## 3. Каналы мониторинга (Channels)

### GET /api/channels
* **Описание**: Получение списка каналов Telegram, добавленных администратором для непрерывного фонового мониторинга ключевых слов.
* **Ответ**: `Array<ChatChannel>`
  ```json
  [
    {
      "id": "chat-1",
      "name": "Анапа Топливо Chat",
      "keywords": "бензин, дт, заправка, 95",
      "excludeFromAnalysis": false
    }
  ]
  ```

### POST /api/channels
* **Описание**: Добавление нового канала в список отслеживания.
* **Тело запроса**:
  ```json
  {
    "name": "Анапа ДТ и Бензин Обсуждения",
    "keywords": "дизель, подвезли, очередь, татнефть"
  }
  ```
* **Ответ**: `{ "status": "ok", "channel": ChatChannel, "channels": Array<ChatChannel> }`

### POST /api/channels/delete
* **Описание**: Удаление канала из списка мониторинга.
* **Тело запроса**: `{ "id": "chat-1" }`
* **Ответ**: `{ "status": "ok", "channels": Array<ChatChannel> }`

### POST /api/channels/toggle-analysis
* **Описание**: Включение или исключение канала из автоматического NLP-анализа сообщений (например, если канал временно переполнился спамом).
* **Тело запроса**: `{ "id": "chat-1" }`
* **Ответ**: `{ "status": "ok", "channels": Array<ChatChannel> }`

## 4. Прямые отчеты водителей (Feedback Bot Reports)

### POST /api/reports
* **Описание**: Имитация входящего сигнала обратной связи от бота Telegram (FeedBack Bot) или водительского приложения.
* **Логика подтверждения**: Если один и тот же отчет (определенный тип топлива и его статус) по конкретной АЗС подтверждается от **двух и более уникальных** подписчиков в течение последних 10 минут, система автоматически переключает статус АЗС в базе. Если это первый сигнал, система запускает внеочередной ИИ-анализ и ожидает повторного подтверждения от других пользователей.
* **Тело запроса**:
  ```json
  {
    "stationId": "station-real-1",
    "fuelType": "DT",
    "type": "unloading",
    "queueLength": "medium",
    "source": "report_bot"
  }
  ```
* **Ответ**:
  ```json
  {
    "status": "ok",
    "station": GasStation,
    "message": ChatMessage,
    "isUpdated": true,
    "matchCount": 2,
    "uniqueSenders": ["Водитель #1", "Водитель #2"]
  }
  ```

## 5. Подписчики (Subscribers / Drivers)

### GET /api/subscribers
* **Описание**: Получение списка активных пользователей системы (авторов отчетов в Telegram) с их статусами и статистикой активности.
* **Ответ**: `Array<Subscriber>`
  ```json
  [
    {
      "name": "Дмитрий В.",
      "status": "active",
      "messagesCount": 14,
      "lastActive": "2026-07-17T17:15:00Z"
    }
  ]
  ```

### POST /api/subscribers/ban
* **Описание**: Блокировка (бан) или разблокировка пользователя по имени. Сообщения от забаненных авторов автоматически классифицируются ИИ как фейк-спам и не влияют на реальный статус АЗС.
* **Тело запроса**: `{ "name": "Дмитрий В." }`
* **Ответ**: `{ "status": "ok", "subscribers": Array<Subscriber> }`

## 6. Системные настройки, Рекомендации и Сброс

### GET /api/settings
* **Описание**: Получение глобальных системных настроек парсера ИИ, прокси-серверов и адреса внешнего RAG-бэкенда.
* **Ответ**: `SystemSettings`

### POST /api/settings
* **Описание**: Обновление глобальных параметров системы и интеграций.
* **Тело запроса**:
  ```json
  {
    "systemPrompt": "Ты - эксперт-аналитик по топливному рынку...",
    "ragContext": "Данные о резервуарах...",
    "temperature": "0.1",
    "llmModel": "gemini-2.5-flash",
    "apiUrl": "",
    "llmApiKey": "",
    "proxyType": "none",
    "proxyAddress": "",
    "recommendationApiUrl": "https://banner-airplane-yard-relay.trycloudflare.com/recommendation"
  }
  ```
* **Ответ**: `{ "status": "ok", "settings": SystemSettings }`

### GET /api/logs
* **Описание**: Получение журнала системных событий для отладки (логирование NLP, API запросов, блокировок, изменений статусов АЗС).
* **Ответ**: `Array<SystemLog>`

### GET /api/recommendations
* **Описание**: Прокси-эндпоинт. Запрашивает рекомендации у внешнего бэкенда Python по адресу `recommendationApiUrl` (через туннели Cloudflare / TryCloudflare). При недоступности туннеля осуществляет бесшовный локальный fallback на встроенный алгоритм прогнозирования дефицита.
* **Ответ**:
  ```json
  {
    "status": "success",
    "source": "external",
    "data": {
      "recommendations": [ ... ]
    }
  }
  ```

### POST /api/reset
* **Описание**: Мгновенный сброс состояния базы данных (АЗС, сообщений, событий) до демонстрационных дефолтных значений для очистки симулятора.
* **Тело запроса**: `{}`
* **Ответ**: `{ "status": "ok", "stations": [...], "chatMessages": [...], "supplyEvents": [...] }`

### GET /api/backend/code
* **Описание**: Получение структурированного исходного кода Python FastAPI бэкенда для последующего экспорта в виде ZIP-архива.
* **Ответ**: `{ "main": "from fastapi import ...", "database": "...", "parser": "...", "schemas": "..." }`
